## Knowledge Stream Curator

Агентная система, которая автоматически превращает поток разрозненных ресурсов (статьи, Telegram‑посты, сайты, GitHub‑репозитории, видео) в структурированную базу знаний с возможностью интеллектуального поиска и подбора материалов.

---

## Проблема

Разработчики, исследователи и команды постоянно сталкиваются с полезными материалами:

* статьи
* посты в Telegram
* GitHub‑репозитории
* образовательные ресурсы
* сайты с инструментами

Обычно такие материалы сохраняются в закладки, "избранное" или заметки.

Со временем формируется **неструктурированный архив**, в котором:

* сложно ориентироваться
* сложно повторно использовать знания
* невозможно быстро подобрать ресурсы по теме

В результате значительная часть найденных материалов фактически теряется.

---

## Решение

**Knowledge Stream Curator** — агентная система, которая:

1. принимает поток контента из разных источников
2. извлекает текст и метаданные
3. очищает и нормализует информацию
4. автоматически определяет тему и тип ресурса
5. извлекает ключевые идеи
6. сохраняет информацию в структурированную базу знаний

После обработки пользователи могут делать запросы к базе знаний.

Пример запроса:

> "Подбери ресурсы, объясняющие diffusion models"

Ответ:

* статьи
* посты
* визуализации
* репозитории

с краткими описаниями и ссылками.

---

## Что демонстрирует PoC

На демо система показывает:

1. Добавление ресурсов

   * Telegram пост
   * ссылка на сайт

2. Автоматическую обработку

   * извлечение текста
   * определение темы
   * генерацию краткого описания
   * извлечение тегов

3. Сохранение в базу знаний

4. Поиск и подборку ресурсов по запросу пользователя.

---

## Out of scope

PoC **не включает**:

* полноценный персональный ассистент
* автоматическое построение курсов
* сложные рекомендательные алгоритмы
* обработку больших корпоративных архивов

---

## Пример пользовательского сценария

Пользователь отправляет системе ссылку:

```
https://example.com/diffusion-models-tutorial
```

Система:

1. извлекает текст
2. определяет тему (diffusion models)
3. генерирует краткое описание
4. присваивает теги
5. сохраняет ресурс

Позже пользователь делает запрос:

> "Подбери материалы по diffusion models"

Система возвращает релевантную подборку.
